# オーダー

## 概要

バドミントン団体戦において、各ゲームに出場するペアを事前に決めること。
試合開始前にオーダーを提出し、相手チームと交換するのが正式な手順。

## このアプリでの扱い

- **対象**: 団体戦（`TeamMatch`）のみ。個人戦（`League`）にはオーダーの概念がない。
- **入力タイミング**: 試合結果を入力する前にオーダーを入力する。オーダーが未入力の場合、結果入力は不可。
- **削除時の注意**: オーダーを削除すると、紐づく試合結果も合わせて削除される。

## 判定ロジック

- **オーダー済** (`hasOrder`): いずれかのゲームにペア選択が入力されている
- **未入力**: 全ゲームでペアが未選択

## UI 上の表示

| 状態 | 表示 |
|------|------|
| オーダー済 | 「オーダー済」バッジ表示 |
| 未入力 | バッジなし |

対戦表ヘッダーには「オーダー済: ○件 / 結果入力済: ○件」と件数を表示する。

## 用語対応
